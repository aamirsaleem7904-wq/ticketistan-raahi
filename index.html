Raahi v3 · UX journey logic — why each screen is designed this way
Price Freeze on listing (S3) — Decision belongs at the point of fare comparison, not at checkout. User selects per flight card.
CFAR as hero (S4) — Cancel Anytime gets its own moment, full-width, with stats. Not buried in a toggle list. This is the highest-value fintech product.
Clean checkout (S7) — No re-offer of PF (already decided). CFAR shown as confirmed active. Focus is pure payment completion.
Fixed confirmation (S8) — Success banner lives inside scrollable content, never overlaps the app header. Header stays consistent throughout.
Bottom nav on all 8 screens — Home, Search, Trips, Wallet, Profile. Active state shown per screen context. Makes it feel like a real app.
Safar Points continuous — Progress bar visible from S1 through S8. Tier unlock in confirmation.
Urdu-first input — RTL user bubbles with Noto Nastaliq. Raahi responds bilingually. Conversational throughout.
Disruption Cover secondary — Sits below CFAR hero. PIA 55% on-time cited. Toggle mechanic, not hard sell.
